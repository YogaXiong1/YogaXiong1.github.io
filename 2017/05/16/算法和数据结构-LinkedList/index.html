算法和数据结构-LinkedList

2017-05-16 • 标签: Swift, 数据结构

用Swift实现了单链表(singly linked list)，实现了简单的链表操作。简单介绍了双向链表及其应用。

单链表

链表的基本概念就不赘述了。这里实现的链表如图，没有定义尾节点，把第一个节点叫头节点。

+--------+    +--------+    +--------+    +--------+
|        |    |        |    |        |    |        |
| node 0 |--->| node 1 |--->| node 2 |--->| node 3 |---> nil
|        |    |        |    |        |    |        |
+--------+    +--------+    +--------+    +--------+
定义节点

节点包括一个值，和其指向的下个节点。

这里使用了范型T。同时，为了方便显示节点信息，我们实现下CustomStringConvertible协议。

定义链表

首先给节点ListNode<T>取了个别名Node，类里的ListNode<T>都可以用Node代替。

格式化输出为[ node => node => node => nil ]

链表的节点个数

从头节点开始遍历计算。复杂度为O(n)。

最后一个节点

同样也是遍历到最后一个节点。复杂度为O(n)。

按照下标获取节点

定义一个游标cursor = index，每次循环自减1，直到 cursor = 0。这里要注意边界情况。

增加一个节点
  • 找最后一个节点，复杂度是O(n)。
  • 让最后一个节点的next指向增加的node，即lastNode.next = node，复杂度是O(1)。
  • 总的复杂度为O(n)。
按照下标插入一个节点
  • 找到下标节点的前一个节点。最好的情况是在头部，复杂度为O(1)，最坏的情况在尾部，复杂度为O(n)。
  • 让最后一个节点的next指向增加的node，即lastNode.next = node，复杂度是O(1)。
  • 总的复杂度为O(n)。

    插入的数据与顺序不大的话，尽量在头部插入，这样更快。

按照下标删除一个节点
  • 找到下标节点和它的前一个节点。
  • 让前一个节点的next指向下标节点的next，下标节点的next设为nil。
  • 总的复杂度为O(n)。

双向链表及其应用

一般双链表结构。

         +--------+    +--------+    +--------+    +--------+
head --->|        |--->|        |--->|        |--->|        |---> nil
         | node 0 |    | node 1 |    | node 2 |    | node 3 |
 nil <---|        |<---|        |<---|        |<---|        |<--- tail
         +--------+    +--------+    +--------+    +--------+

节点可以这样定义。

next和previous不能同时为strong，会导致循环引用。
